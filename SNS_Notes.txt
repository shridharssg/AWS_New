✅ SNS
--------------------------------------------------

1️⃣ WHAT IS SNS
• SNS = Simple Notification Service
• Event broadcasting service
• Uses Publish–Subscribe model

✔ One publisher → many subscribers
✔ Completely decoupled system

--------------------------------------------------

2️⃣ WHY SNS IS USED
• One event triggers multiple independent actions
• Avoids API complexity
• No need to push to multiple queues manually

✔ Extensible architecture
✔ Future-proof system

--------------------------------------------------

3️⃣ CORE FLOW
Publisher → SNS Topic → Subscribers

✔ Publisher does NOT know subscribers
✔ Subscribers react independently

--------------------------------------------------

4️⃣ SNS vs SQS
• SNS = Event broadcasting (fanout)
• SQS = Work queue (processing)

👉 RULE:
✔ SNS announces events
✔ SQS processes work

--------------------------------------------------

5️⃣ SNS + SQS (BEST PRACTICE)
API / Lambda
  ↓
SNS Topic
  ↓
SQS Queues
  ↓
Worker Lambdas

✔ SNS → Fanout
✔ SQS → Reliability (Retry, DLQ)
✔ Lambda → Processing

--------------------------------------------------

6️⃣ SUBSCRIPTION TYPES
✅ SQS (MOST COMMON)
✅ Lambda
✅ Email / SMS
✅ HTTP endpoints

--------------------------------------------------

7️⃣ MESSAGE FILTERING
• SNS can route messages based on attributes
• Same topic → different behavior

✔ No change in publisher code
✔ Clean architecture

--------------------------------------------------

8️⃣ WHEN TO USE SNS
• One event → many actions
• Need loose coupling
• Want to add consumers without API change

--------------------------------------------------

✅ DETAILED EXPLANATION
--------------------------------------------------

1️⃣ WHY SNS EXISTS (REAL PROBLEM)

Example:
POST /orders/create

When order is created:
• Send Email
• Send SMS
• Notify Admin
• Generate Invoice
• Send Analytics

❌ Without SNS:
• API must call all services
• API knows all downstream systems
• Every new requirement → code change

Problems:
❌ Tight coupling
❌ Hard to scale
❌ Hard to maintain

✅ SNS SOLUTION:
✔ API sends ONE event
✔ SNS handles distribution

--------------------------------------------------

2️⃣ WHAT SNS REALLY IS

SNS = EVENT BROADCASTER

👉 Simple meaning:
• You announce “Something happened”
• Others decide what to do

✔ Publisher does not control consumers
✔ Consumers act independently

--------------------------------------------------

3️⃣ CORE MENTAL MODEL (LOCK THIS)

Publisher → SNS Topic → Subscribers

Publisher:
• API / Lambda

SNS Topic:
• Communication channel

Subscribers:
• SQS / Lambda / Email etc.

✔ No direct connection between them

--------------------------------------------------

4️⃣ HOW SNS WORKS (STEP BY STEP)

Step 1: Publisher sends event
  publish("OrderCreated")

Step 2: SNS receives message
  • Identifies subscribers

Step 3: SNS fans out
  • Creates copies
  • Sends to all subscribers

Step 4: Subscribers process
  • Independently
  • In parallel

✔ No blocking
✔ No dependency

--------------------------------------------------

5️⃣ REAL ARCHITECTURE (PRODUCTION)

SNS Topic (OrderCreated)
   ├─ EmailQueue → EmailWorker
   ├─ InvoiceQueue → InvoiceWorker
   ├─ AnalyticsQueue → AnalyticsWorker
   └─ AdminQueue → AdminWorker

✔ Each system:
• Independent
• Scalable
• Fault-isolated

--------------------------------------------------

6️⃣ WHY SNS + SQS IS BEST

❌ SNS → Lambda directly:
• Limited retry
• No buffering
• Risk of data loss

✅ SNS → SQS → Lambda:
• SQS stores message
• Retry supported
• DLQ supported
• Handles load spikes

👉 GOLDEN RULE:
SNS = Fanout
SQS = Reliability

--------------------------------------------------

============================================================Example======================

When order is placed → send email using SNS

Flow:
Lambda (Order Placed) → SNS → Email

============================================================
STEP 1: CREATE SNS TOPIC
============================================================

1. Go to AWS Console
2. Search SNS
3. Click "Topics"
4. Click "Create topic"
5. Select "Standard"
6. Name: order-topic
7. Click "Create topic"

✅ Copy Topic ARN

============================================================
STEP 2: ADD EMAIL SUBSCRIBER
============================================================

1. Open SNS Topic (order-topic)
2. Click "Create subscription"
3. Protocol → Email
4. Endpoint → your-email@example.com
5. Click "Create subscription"

✅ Check your email and click "Confirm subscription"

============================================================
STEP 3: CREATE LAMBDA (ORDER PLACED)
============================================================

1. Go to AWS Lambda
2. Click "Create function"
3. Select "Author from scratch"
4. Function name: order-lambda
5. Runtime: Node.js 18
6. Click "Create function"

------------------------------------------------------------
Add Code:
------------------------------------------------------------

import { SNSClient, PublishCommand } from "@aws-sdk/client-sns";

const sns = new SNSClient({ region: "ap-south-1" });

export const handler = async () => {

  const order = {
    orderId: "101",
    userEmail: "customer@example.com"
  };

  const message = `Order placed successfully. Order ID: ${order.orderId}`;

  await sns.send(new PublishCommand({
    TopicArn: process.env.TOPIC_ARN,
    Message: message,
    Subject: "Order Confirmation"
  }));

  return "Email sent";
};

------------------------------------------------------------
Add Environment Variable:
------------------------------------------------------------

Key: TOPIC_ARN
Value: Paste SNS Topic ARN

------------------------------------------------------------
Permissions:
------------------------------------------------------------

Attach policy: AmazonSNSFullAccess

============================================================
STEP 4: TEST
============================================================

1. Open Lambda (order-lambda)
2. Click "Test"

============================================================
EXPECTED RESULT
============================================================

1. Lambda runs
2. SNS receives message
3. SNS sends email
4. You receive email notification
