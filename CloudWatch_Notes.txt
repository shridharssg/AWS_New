✅ AWS CLOUDWATCH (MONITORING & OBSERVABILITY)
==================================================

✅ WHAT IS CLOUDWATCH
--------------------------------------------------
CloudWatch is an AWS service for monitoring and observability

Helps answer:
	• What happened
	• How bad it is
	• When to act

--------------------------------------------------

✅ CLOUDWATCH CORE COMPONENTS
--------------------------------------------------

✅ CloudWatch Logs
	• Stores application logs (console.log, errors)
	• Auto-enabled for Lambda
	• Used for debugging failures
	👉 Logs = What happened

✅ CloudWatch Metrics
	• Numerical system data
	• Examples:
		• Lambda invocations
		• Errors
		• Duration
		• API latency
	👉 Metrics = How often & how bad

✅ CloudWatch Alarms
	• Monitor metrics
	• Trigger alerts
	• Send via SNS (email, SMS)
	👉 Alarms = When to react

--------------------------------------------------

✅ ADDITIONAL IMPORTANT FEATURES (MISSING PART)
--------------------------------------------------

✅ Log Retention
	• Set how long logs are stored
	• Avoid high cost

✅ CloudWatch Log Insights
	• Query logs using SQL-like queries
	• Faster debugging

✅ Dashboards
	• Visualize metrics in one place
	• Useful for production monitoring

✅ CloudWatch Events / EventBridge
	• Trigger actions based on events
	• Example:
		• Auto start Lambda on schedule

✅ X-Ray Integration
	• Trace request flow
	• Find latency bottlenecks

--------------------------------------------------

✅ COMMON SERVICES MONITORED

• Lambda

✅ DEBUGGING USING CLOUDWATCH (STEP-BY-STEP)
==================================================

STEP 1: Identify Symptom
	• API failed?
	• Timeout?
	• Data missing?

STEP 2: Check API Response
	• 4XX → Client issue
	• 5XX → Backend issue

STEP 3: Check Lambda Logs
	✅ Logs present → Lambda executed
	❌ No logs → blocked earlier

STEP 4: Analyze Logs
	• Errors
	• Stack trace
	• Input data

STEP 5: Use Request ID
	• Trace request end-to-end
	• Log requestId in Lambda

STEP 6: Check Metrics
	• Errors spike → bug
	• Duration ↑ → latency issue
	• Throttles → scaling issue

STEP 7: Check Downstream
	• DynamoDB
	• SQS
	• SNS

STEP 8: Check Alarms
	• Alerts already triggered?

STEP 9: Fix & Monitor
	• Deploy fix
	• Observe logs + metrics

--------------------------------------------------

✅ BEST PRACTICES (IMPORTANT ADDITION)
--------------------------------------------------
• Always add meaningful logs
• Use structured logging (JSON)
• Set log retention policy
• Create alarms for critical metrics
• Monitor error rate + latency
• Use dashboards for visibility

CloudWatch monitors AWS applications using logs, metrics, and alarms for debugging and alerting
