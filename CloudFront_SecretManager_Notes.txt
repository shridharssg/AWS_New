✅ CLOUDFRONT & SECRETS MANAGER (AWS)
==================================================

✅ PART 1 — AMAZON CLOUDFRONT
==================================================

✅ WHY CLOUDFRONT IS NEEDED (REAL SCENARIO)
--------------------------------------------------
Scenario:
	• Users from different countries
	• Same data requested repeatedly
	• Frontend loads images/APIs

❌ Without CloudFront:
	• Every request hits backend
	• High latency
	• Increased backend load
	• Higher cost

✅ Solution:
	👉 CloudFront caches data near users

--------------------------------------------------

✅ WHAT IS CLOUDFRONT
--------------------------------------------------
CloudFront = CDN (Content Delivery Network)

Purpose:
	• Cache responses near users
	• Reduce backend calls
	• Improve speed

--------------------------------------------------

✅ ARCHITECTURE POSITION
--------------------------------------------------
User
 ↓
CloudFront
 ↓
API Gateway / S3
 ↓
Lambda / ECS

✅ Always sits in FRONT

--------------------------------------------------

✅ USE CASES
--------------------------------------------------
• Static assets (JS, CSS, Images)
• Frontend hosting
• API caching (GET requests)
• Traffic spike protection

--------------------------------------------------

✅ WHAT CLOUDFRONT DOES NOT DO
--------------------------------------------------
❌ No business logic
❌ No authentication
❌ Does not replace API Gateway

✅ It only caches & accelerates

--------------------------------------------------

✅ SIMPLE FLOW
--------------------------------------------------
Without:
	User → API → Lambda

With CloudFront:
	User → CloudFront → API → Lambda

If cached:
	✅ Lambda not called
	✅ Faster response
	✅ Lower cost

--------------------------------------------------

✅ WHEN TO USE CLOUDFRONT
--------------------------------------------------
• Global users
• High read traffic
• Static content
• API response caching

--------------------------------------------------

✅ ADDITIONAL IMPORTANT POINTS
--------------------------------------------------
• Uses Edge Locations (global)
• Supports HTTPS (SSL termination)
• Integrates with WAF (security)
• Caching rules (TTL control)

--------------------------------------------------

✅ PART 2 — AWS SECRETS MANAGER
==================================================

✅ WHY SECRETS MANAGER IS NEEDED
--------------------------------------------------
Backend needs:
	• DB password
	• API keys
	• Third-party credentials

❌ Risks:
	• Hardcoding in code
	• Storing in Git

✅ Solution:
	👉 Secure secret storage

--------------------------------------------------

✅ WHAT IS SECRETS MANAGER
--------------------------------------------------
Secrets Manager = Secure vault for secrets

Features:
	• Stores credentials
	• Encrypts data
	• IAM-based access

--------------------------------------------------

✅ ARCHITECTURE FLOW
--------------------------------------------------
Lambda / ECS
 ↓ (IAM Access)
Secrets Manager
 ↓
Returns secret (runtime)

✅ No hardcoding

--------------------------------------------------

✅ NODE FLOW
--------------------------------------------------
• Code requests secret at runtime
• IAM authenticates
• Secret returned securely

--------------------------------------------------

✅ SECRETS vs ENV VARIABLES
--------------------------------------------------
Env Variables:
	• Plain text
	• Less secure
	• Hard to rotate

Secrets Manager:
	• Encrypted
	• Highly secure
	• Easy rotation

--------------------------------------------------

✅ WHAT SECRETS MANAGER DOES NOT DO
--------------------------------------------------
❌ Does not run code
❌ Does not manage users
❌ Does not replace database

✅ Only secure storage

--------------------------------------------------

✅ ADDITIONAL IMPORTANT POINTS (MISSING)
--------------------------------------------------
• Supports automatic rotation
• Integrates with RDS for password rotation
• Versioning of secrets
• Audit via CloudTrail

--------------------------------------------------

✅ INTERVIEW TAKEAWAY
--------------------------------------------------
CloudFront:
	• Improves performance
	• Caches content near users

Secrets Manager:
	• Improves security
	• Removes secrets from code

--------------------------------------------------

✅ ONE-LINE SUMMARY
--------------------------------------------------
CloudFront improves performance by caching near users,
and Secrets Manager improves security by protecting credentials.
